The most valuable tools in a workflow are often small enough to understand at a glance. Their advantage comes from removing a repeated interruption.

## Automate the repeated edge

Look for a task that is easy but frequent: setting up a directory, checking a file format, or collecting the same diagnostic information.

## Keep the interface obvious

A tiny tool should have a tiny contract. Prefer one memorable command, useful defaults, and output that explains what happened.

## Let use shape the tool

Start with the narrowest useful version. Once it becomes part of real work, the next improvement will be much easier to see.
